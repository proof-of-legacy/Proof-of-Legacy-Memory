Algorithm

How PoLM works

score = (1 / latency_ns) × boost × thread_penalty
boost = (latency_ns / 1000)^0.8  ·  DDR2 at 3800ns → ~11× effective boost
01
Build the Memory DAG
Each mining epoch seeds a 256 MB pseudorandom buffer from the chain state. It defeats CPU cache entirely — every access is a real DRAM read.
02
Random memory walk
100,000 sequential steps, each depending on the last. Cannot be parallelized. The bottleneck is purely DRAM latency — physics, not software.
03
Legacy hardware wins
Higher latency = higher boost. DDR2 at ~3800ns gets ~10× while DDR4 at ~1060ns gets ~1×. The gap between generations: ~2.5× — vs 100× in SHA-256.
Get started

Start mining in 3 commands

# Linux / macOS

# Clone and install
git clone https://github.com/proof-of-legacy/Proof-of-Legacy-Memory.git
cd Proof-of-Legacy-Memory && bash scripts/install.sh

# Start mining (auto-detects your RAM type)
python3 polm.py miner http://node1.polm.com.br:6060 YOUR_ADDRESS DDR2
